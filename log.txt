§5. DEPLOY STAGING  — LỖI PULL SECRET & SỬA
================================================================================

$ ./scripts/deploy-do.sh ../app-repos/notes-app --env staging \
    --registry registry.digitalocean.com/idp-notes-thanhnt/notes-app --lb-ip 104.248.96.140
  -> Render 8 manifest + 1 secret; tạo ns notes-app-staging; apply toàn bộ OK.
  -> NHƯNG bước tạo pull secret DOCR LỖI:
     "the namespace from the provided object \"kube-system\" does not match the
      namespace \"notes-app-staging\""

  Nguyên nhân: `doctl registry kubernetes-manifest` gắn CỨNG metadata.namespace=kube-system
  trong manifest secret -> apply vào namespace khác bị từ chối. Hệ quả: pod ImagePullBackOff
  (không có secret kéo image từ DOCR private).

  SỬA (deploy-do.sh): xoá field namespace trước khi apply:
     doctl registry kubernetes-manifest | yq 'del(.metadata.namespace)' \
       | kubectl apply -n "$NAMESPACE" -f -

$ (áp lại pull secret đã sửa cho staging) -> secret/registry-idp-notes-thanhnt created
$ kubectl -n notes-app-staging delete pod <backend/frontend đang lỗi>  (ép pull lại)
$ kubectl -n notes-app-staging get pods
  -> backend 1/1, backend-db-0 1/1, frontend 1/1  (Running)

LỖI & SỬA (phần 2):

     -> kubectl apply thường vượt giới hạn annotation last-applied cho CRD lớn.

     khi secrets đã có. (Nên vá: đặt secret trước khi push nội dung — ghi chú trong repo.)
  6) [Workflow cũ] ci.yaml + orchestrator.yaml gốc (self-hosted/Harbor) cũng bị trigger,
     treo vô hạn. SỬA: gh workflow disable ci / orchestrator + cancel run.
  7) [ArgoCD sync] Deployment backend/frontend từ lần deploy TAY (Phần 1) có
     selector.app.kubernetes.io/instance khác hash với bản render orchestrator ->
     spec.selector IMMUTABLE, sync fail. SỬA: xoá 2 Deployment cũ để ArgoCD tạo lại
