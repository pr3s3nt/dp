1) platform-repo/score/provisioners/do.provisioners.yaml
   - Cùng giao diện Score với onprem/cloud; 7 loại resource:
     postgres/mysql/mongodb/redis/app-config/route/service.

   - route: Ingress class nginx (ingress-nginx trên DOKS tự xin 1 DO LoadBalancer).

2) platform-repo/score/patches/do-staging.tpl  (replicas 1, resources nhỏ)
   platform-repo/score/patches/do-prod.tpl     (replicas 2, resources lớn hơn)

   - Quy tắc giữ nguyên: bỏ qua manifest label component=datastore (không set

   - prod để replicas=2 (không phải 3 như EKS) để vừa cụm test 1 node mà vẫn KHÁC staging.

3) platform-repo/scripts/deploy-do.sh
   - Render mỗi service bằng: score-k8s init --provisioners do.provisioners.yaml
       --patch-templates do-<env>.tpl ; rồi score-k8s generate (override image về DOCR).
   - 2 env TÁCH NHAU BẰNG NAMESPACE: notes-app-staging vs notes-app-prod.
   - Override host route theo env bằng nip.io (<app>-<env>.<LB_IP>.nip.io) vì 2 env
     chung 1 cụm/1 ingress -> phải khác host để không tranh traffic.
   - Tách Secret khỏi manifest (secret KHÔNG vào git; apply create-if-missing).
   - Tạo namespace + pull secret DOCR + apply.

§5. DEPLOY STAGING  — LỖI PULL SECRET & SỬA
================================================================================

$ ./scripts/deploy-do.sh ../app-repos/notes-app --env staging \
    --registry registry.digitalocean.com/idp-notes-thanhnt/notes-app --lb-ip 104.248.96.140
  -> Render 8 manifest + 1 secret; tạo ns notes-app-staging; apply toàn bộ OK.
  -> NHƯNG bước tạo pull secret DOCR LỖI:
     "the namespace from the provided object \"kube-system\" does not match the
      namespace \"notes-app-staging\""

  Nguyên nhân: `doctl registry kubernetes-manifest` gắn CỨNG metadata.namespace=kube-system
  trong manifest secret -> apply vào namespace khác bị từ chối. Hệ quả: pod ImagePullBackOff
  (không có secret kéo image từ DOCR private).

  SỬA (deploy-do.sh): xoá field namespace trước khi apply:
     doctl registry kubernetes-manifest | yq 'del(.metadata.namespace)' \
       | kubectl apply -n "$NAMESPACE" -f -

$ (áp lại pull secret đã sửa cho staging) -> secret/registry-idp-notes-thanhnt created
$ kubectl -n notes-app-staging delete pod <backend/frontend đang lỗi>  (ép pull lại)
$ kubectl -n notes-app-staging get pods
  -> backend 1/1, backend-db-0 1/1, frontend 1/1  (Running)
